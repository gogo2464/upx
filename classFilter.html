Detailed Description

A filter is a reversible operation that modifies a given block of memory.

A filter can fail and return false. In this case the buffer must be unmodified (or otherwise restored).

If a filter fails and somehow cannot restore the block it must call throwFilterException() - this will cause the compression to fail.

Unfilters throw exceptions in case of errors.

The main idea behind filters is to convert relative jumps and calls to absolute addresses so that the buffer compresses better.
